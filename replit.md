# SIGAA API

## Overview
An unofficial Node.js/TypeScript web scraping library for SIGAA (Sistema Integrado de Gestão de Atividades Acadêmicas), a Brazilian academic management system used by federal institutes and universities (IFSC, IFFar, UFPB, UNB).

**Note:** This project is archived — the original author has graduated and no longer maintains it.

## Architecture
- **Type:** TypeScript library (no frontend or web server)
- **Language:** TypeScript → compiled via Babel to JavaScript
- **Build output:** `dist/` directory
- **Main entry:** `dist/sigaa-all-types.js`

## Key Files
- `src/` — TypeScript source code
- `dist/` — Compiled JavaScript output (generated by `npm run build`)
- `examples/` — Usage examples
- `babel.config.json` — Babel transpilation configuration
- `tsconfig.json` — TypeScript config (relaxed for compatibility with newer package versions)

## Build System
Uses **Babel** for transpilation (not `tsc`), which skips type-checking. This was necessary because:
- `formdata-node` was upgraded from v3 to v6 (breaking API changes)
- `cheerio` was upgraded from v1.0.0-rc.10 to v1.2.0 (type API changes)
- `typescript` was upgraded from v4 to v5 (stricter catch variable types)

### Fixes Applied During Setup
1. Replaced `formdata-node` with `form-data` package (has `append()`, `getHeaders()`, `getBuffer()`)
2. Changed `cheerio.Root` type to `CheerioAPI` from newer cheerio
3. Changed `cheerio.Element` / `cheerio.Cheerio` to `any` types
4. Removed `normalizeWhitespace` option from `$load()` (removed in newer cheerio)
5. Disabled TypeScript `strict`, `strictNullChecks`, `isolatedModules` in tsconfig
6. Removed `tsc --emitDeclarationOnly` from build script (type errors from package updates)
7. Installed missing `@babel/plugin-proposal-class-properties`

## Dependencies
- `cheerio` — HTML parsing/scraping
- `form-data` — Multipart form handling
- `he` — HTML entity decoding
- `iconv-lite` — Character encoding
- `lodash` — Utility functions
- `source-map-support` — Source maps for debugging

## Workflow
- **"Start application"** — Runs `npm run build` (console output)

## Usage
```bash
npm run build  # Build the library to dist/
```

Then use in a Node.js project:
```js
const { Sigaa } = require('./dist/sigaa-all-types.js');
const sigaa = new Sigaa({ url: 'https://sigaa.ifsc.edu.br', institution: 'IFSC' });
```

See `examples/` for usage samples.
