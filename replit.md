# SIGAA API

## Overview
An unofficial Node.js/TypeScript web scraping library for SIGAA (Sistema Integrado de Gestão de Atividades Acadêmicas), a Brazilian academic management system used by federal institutes and universities (IFSC, IFFar, UFPB, UNB).

**Note:** This project is archived — the original author has graduated and no longer maintains it.

## Architecture
- **Type:** TypeScript library (no frontend or web server)
- **Language:** TypeScript → compiled via Babel to JavaScript
- **Build output:** `dist/` directory
- **Main entry:** `dist/sigaa-all-types.js`

## Key Files
- `src/` — TypeScript source code
- `dist/` — Compiled JavaScript output (generated by `npm run build`)
- `examples/` — Usage examples
- `babel.config.json` — Babel transpilation configuration
- `tsconfig.json` — TypeScript config (relaxed for compatibility with newer package versions)

## Build System
Uses **Babel** for transpilation (not `tsc`), which skips type-checking. This was necessary because:
- `formdata-node` was upgraded from v3 to v6 (breaking API changes)
- `cheerio` was upgraded from v1.0.0-rc.10 to v1.2.0 (type API changes)
- `typescript` was upgraded from v4 to v5 (stricter catch variable types)

### Fixes Applied During Setup
1. Replaced `formdata-node` with `form-data` package (has `append()`, `getHeaders()`, `getBuffer()`)
2. Changed `cheerio.Root` type to `CheerioAPI` from newer cheerio
3. Changed `cheerio.Element` / `cheerio.Cheerio` to `any` types
4. Removed `normalizeWhitespace` option from `$load()` (removed in newer cheerio)
5. Disabled TypeScript `strict`, `strictNullChecks`, `isolatedModules` in tsconfig
6. Removed `tsc --emitDeclarationOnly` from build script (type errors from package updates)
7. Installed missing `@babel/plugin-proposal-class-properties`

## HTTP Layer — Cloudflare Bypass (puppeteer-real-browser)

The original raw HTTPS/axios HTTP layer has been replaced with a real browser approach using `puppeteer-real-browser`. This allows the library to bypass Cloudflare bot protection that SIGAA institutions have deployed.

### New/Modified Session Files
| File | Role |
|------|------|
| `src/session/sigaa-browser.ts` | `SigaaBrowserImpl` — manages browser lifecycle, navigation, form submission, file download, Cloudflare wait. Exports `CloudflareError` and `BrowserNotInitializedError`. |
| `src/session/sigaa-http.ts` | `SigaaHTTP` — same public interface but backed by browser instead of raw HTTPS. `SigaaBrowserImpl` is an optional second constructor arg. |
| `src/session/sigaa-http-factory.ts` | `SigaaHTTPFactory` — now takes `SigaaBrowserImpl` as 4th constructor arg and passes it to `SigaaHTTP`. |
| `src/session/sigaa-page-navigator.ts` | `SigaaPageNavigator` — JSF navigation helpers (`clickMenuItemByText`, `submitJSFForm`, `checkSessionExpired`). |
| `src/session/login/sigaa-login-ifsc.ts` | Browser-based login for IFSC (uses `user.login`/`user.senha`, navigates `/sigaa/verTelaLogin.do`). |
| `src/session/login/sigaa-login-ufpb.ts` | Browser-based login for UFPB (uses `form:login`/`form:senha`, navigates `/sigaa/logon.jsf`). |
| `src/session/login/sigaa-login-unb.ts` | Browser-based login for UNB (same pattern as IFSC). |
| `src/sigaa-main.ts` | Creates `SigaaBrowserImpl` and passes it through; accepts optional `browser?: SigaaBrowserOptions` constructor option. |

### Key Architecture Notes
- **`headless: false` is MANDATORY** for puppeteer-real-browser — browser must be visible (uses Xvfb in headless server environments).
- **Cookie handling is native** to the browser — `SigaaHTTPSession.afterHTTPOptions` may still add a Cookie header but it's ignored by the browser.
- **Fake HTTP metadata** — browser-built `SigaaPage` objects use `statusCode: 200` and `headers: {}` since the browser doesn't expose raw HTTP responses.
- **Redirects are automatic** — `followAllRedirect()` is a no-op since the browser follows redirects natively.
- **Login flow** — `http.get(loginPath)` triggers browser navigation, then the raw puppeteer page is used for `page.evaluate()` to fill forms and click buttons. Never use `page.realClick()` (ghost-cursor incompatibility) or `keyboard.type()` (unreliable).
- **skipIntermediatePages** — only clicks **visible** elements with "Continuar" text (checks `display`, `visibility`, `opacity`, `offsetParent`). Prevents false clicks on hidden rotator links in the portal.
- **submitForm(postValues, actionUrl?)** — always receives the target action URL from `SigaaHTTP`. Creates a temporary hidden form with the action URL and all POST values, ensuring correct submission even when the browser has navigated away from the original page. Falls back to finding an existing form on the page if no `actionUrl` is provided.
- **Navigation wait strategy** — uses `waitUntil: 'domcontentloaded'` (not `load`/`networkidle2`) for both `navigate()` and `submitForm()` to avoid hanging on heavy SIGAA pages with continuous network activity (analytics, clocks, AJAX).
- **Race condition prevention** — `waitForNavigation` promise is created BEFORE `form.submit()` to avoid missing fast navigation events. The `evaluate()` call is fire-and-forget (`.catch(() => {})`) since the execution context is destroyed when navigation starts.
- **Request stack fix** — `buildSigaaPage()` now receives the original `sessionHttpOptions` (with the correct method, path, headers) instead of rebuilding from the final URL. This ensures `afterSuccessfulRequest` can match and resolve the request stack promise, allowing subsequent POST requests to proceed.

### Account Parser Fixes
- `src/account/sigaa-account-ifsc.ts` `parseStudentHomePage()`: Modern IFSC SIGAA no longer has a `<table>` inside `#perfil-docente` with Matrícula/Curso/Status rows. The parser now falls back to fetching `/sigaa/vinculos.jsf` and calling `parseBondPage()` when the table is missing or incomplete.

## Workflow
- **"Start application"** — Runs `npm run build` (console output)

## Usage
```bash
npm run build  # Build the library to dist/
```

Then use in a Node.js project:
```js
const { Sigaa } = require('./dist/sigaa-all-types.js');
const sigaa = new Sigaa({ url: 'https://sigaa.ifsc.edu.br', institution: 'IFSC' });
```

See `examples/` for individual usage samples, or run the interactive menu:
```bash
node sigaa-menu.js
```
The menu prompts for SIGAA credentials once, then offers numbered options for all features (account info, courses, grades, absences, activities, homework, lessons, news, file downloads). No technical knowledge required.
